# 文件重定向（File Redirector）

> 一个运行于 Android root 环境的自动化文件管理模块，用于解决应用随意将文件写入不规范目录的问题，实现存储空间的规范与整洁。
>
> 不同于常见的 inotifywait 脚本方案，本模块基于命名管道多路复用架构构建：inotify 事件流与看门狗控制信号经由同一 FIFO 统一调度，服务崩溃可无感自愈，规则修改后自动生效，脚本自身更新时通过 exec 原地热重启而非重新拉起进程——空闲时 CPU 占用接近零，无任何轮询。

---

## 环境要求

- Android 设备（已 root）
- KernelSU / Magisk / APatch（任一框架均可）
- 支持 WebUI 将会获得最佳体验

---

## 使用方法

1. 在 KernelSU / Magisk / APatch 管理器中刷入模块 zip 包
2. 重启设备
3. 打开 ROOT 管理器 → 模块 → 点击「文件重定向」的 WebUI 按钮
4. 在规则页面添加重定向或清理规则，保存后自动生效

---

## 核心特性

- **FIFO 多路复用**：inotify 事件与看门狗信号经同一命名管道统一调度，生产端与消费端完全解耦，写入端退出不触发 EOF，服务不因 inotifywait 崩溃而中断
- **无感热重载**：规则文件修改后主循环下一轮自动检测生效，无需重启服务；脚本自身更新时 exec 原地替换进程，PID 不变
- **原子移动 + 降级链**：优先 mv 原子操作；跨分区时自动降级为 cp + 字节校验 + 权限复制 + rm，每步有独立回滚
- **原子文件锁**：同一文件在多事件并发触发下不会被重复处理
- **零轮询**：基于 inotify 事件驱动，空闲时 CPU 占用接近零

## 使用场景

### 重定向规则
监控指定目录，将匹配文件自动移动到目标目录。适用于：
- 将 App 随意保存到公共目录的图片/视频归类到专属文件夹
- 替代「存储空间隔离」的导出功能，避免相册有时扫描不到的问题
- 整理截图、分享缓存等散落在各处的文件

### 清理规则
监控指定目录，将匹配文件直接删除。适用于：
- 清理部分应用保存在公共存储空间的垃圾文件/文件夹

---

## 规则配置

### 规则文件格式

```
[监控目录]+[匹配规则]+[目标目录]+[N]
```
**示例：**
```
[/sdcard/DCIM/taplus/taplus_image]+[*]+[/sdcard/Pictures/taplus]+[2]
```

### 注意事项
- 目标目录不能同时是监控目录，防止重定向死循环
- 保留值 `__delete__` 为删除模式功能关键字，请勿手动填写
- 兼容跨文件系统重定向，但此时重定向会失去原子性
